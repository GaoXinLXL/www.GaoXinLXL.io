反射

简单理解，反射是为了解决在运行期，对某个实例一无所知的情况下，如何调用其方法。

  1. class是由JVM在执行过程中，动态加载的。JVM第一次读到一种class类型时，就将其放进内存。
  2. 每加载一种class，JVM就为其创建一个Class类型的实例，并关联起来。只有JVM能创建Class实例。
  3. 一个Class实例包含了该class的所有完整信息。包括：
    1. 类名
    2. 包名
    3. 实现的接口
    4. 所有方法
    5. 所有字段
    6. ……

通过Class实例获取class信息的方法就是“反射(Reflection)”。
以String类为例：当JVM加载String类时，它先将String.class文件放进内存，然后，为String类创建一个Class实例并关联起来。

🔨获得一个class的Class实例的3个方法

  1. 直接由一个class的静态变量class获取
    Class cls = String.class;
  1. 由实例变量，通过其方法getClass()获取
    String s = "asd";
Class cls = s.getClass();
  1. 知道完整类名，可以由静态方法Class.forName()获取
    Class cls = Class.forName("java.lang.String");


Class实例在JVM中是唯一的，所以这3种方法得到是同一个实例。

🔑怎么调用方法？

原代码如下：

String s = "Hello world";
String r = s.substring(6); // "world"


用反射来调用substring方法，如下：

String s = "Hello world";
Method m = String.class.getMethod("substring",int.class);
String r = (String)m.invoke(s,6);
