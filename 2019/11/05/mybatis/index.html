MyBatis

发表于 2019-11-05 更新于 2020-02-17 分类于 笔记

MyBatis 是一款优秀的持久层框架，它支持定制化 SQL、存储过程以及高级映射。MyBatis避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。MyBatis可以使用简单的 XML 或注解来配置和映射原生类型、接口和 Java 的 POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

提前了解

两个配置文件

.XML

  1. 全局配置，一般叫做mybatis-config.xml
  2. 映射配置，一般叫做XxxMapper.xml

这两个配置文件都可以参考MyBatis官方文档，不过需要量身修改

配置文件需知

  1. mybatis-config.xml
<?xml version="1.0" encoding="UTF-8" ?><!DOCTYPE configuration        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"        "http://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration> 
    <environments default="development">  
        <environment id="development">            <transactionManager type="JDBC"/>         
            <dataSource type="POOLED">           
                <property name="driver" value="com.mysql.jdbc.Driver"/>          
                <property name="url" value="jdbc:mysql://localhost:3306/mybatis"/>                <property name="username" value="root"/>          
                <property name="password" value="1234"/>            </dataSource>   
        </environment> 
    </environments>   

    <mappers>       
        <mapper resource="UserMapper.xml"/>  
    </mappers>  

</configuration>

主配置文件重点掌握和的配置

  • <environments>

    跟数据库有关的信息

  • <mappers>

    指向XxxMapper.xml配置文件

  1. XxxMapper.xml
<?xml version="1.0" encoding="UTF-8" ?><!DOCTYPE mapper        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.it.mapper.UserMapper">   
    <select id="selectUser" resultType="com.it.pojo.User">                    select * from user;   
    </select>
</mapper>

可以这样理解，mapper配置文件相当于对dao接口的具体实现

注意这里的id要跟XxxMapper.java接口里的方法名保持一致

MaBatis工具类

为了封装一些每次都要做套路，最好先建立一个工具类MyBatisUtils.java

这里的代码基本不用改变

代码展示

public class MyBatisUtils {  
    private static SqlSessionFactory sqlSessionFactory;  
    static {   
        try {       
            String resource = "mybatis-config.xml";            InputStream inputStream = Resources.getResourceAsStream(resource);            sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);      
        } catch (IOException e) {      
            e.printStackTrace();      
        }    
    }  
    public static SqlSession getSession(){    
        return sqlSessionFactory.openSession();   
    }
}

代码分析

  1. 这个工具类总的来说包含两个部分

    • 生成SqlSessionFactory对象
    • 获取SqlSession对象

  2. 生成SqlSessionFactory对象

    套路，官方文档有：

String resource = "mybatis-config.xml";
InputStream inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
  1. 获取SqlSession对象

这里要自己建立一个静态方法，通过生成的SqlSessionFactory对象的openSession()方法来创建

建立UserMapper接口

存放对数据库要进行的操作，相当于DAO

public interface UserMapper {
    List<User> selectUser();//查询所有用户
}

写UserMapper.xml配置文件

就把它当作是对UserMapper接口的实现

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.it.mapper.UserMapper">
    <select id="selectUser" resultType="com.it.pojo.User">
        select * from user;
    </select>
</mapper>
  • 命名空间=UserMapper接口的位置
  • id=UserMapper接口里的selectUser方法
  • resultType=结果类型的全类名

测试

public class UserMapperTest {
    @Test
    public void selectUser(){
        SqlSession sqlSession = MyBatisUtils.getSession();
        UserMapper mapper = sqlSession.getMapper(UserMapper.class);
        List<User> users = mapper.selectUser();
        for (User user : users) {
            System.out.println(user);
        }
        sqlSession.close();
    }
}

  1. 通过MyBatisUtils工具类获得SqlSession对象

    相当于connection

  2. 由SqlSession对象得到UserMapper接口的”实例对象”

    实际上，并没有接口的实现类，接口的实现是借助配置文件完成的

  3. 调用接口里存在的方法即可

  4. 最后一定要关闭SqlSession

万能的Map

巧用Map

假设，我们的实体类，或者数据库中的表，字段或参数过多，应当考虑使用Map

  1. UserMapper接口：
public interface UserMapper {   
    //巧用Map 
    void addUser(Map<String,Object> map);
}
  1. UserMapper.xml：
<insert id="addUser" parameterType="map">
    insert into mybatis.user values (#{uid},#{uname},#{upwd})
</insert>
  1. 测试
public void addUser(){
        SqlSession sqlSession = MyBatisUtils.getSession();
        UserMapper mapper = sqlSession.getMapper(UserMapper.class);
        Map<String,Object> map = new HashMap<String, Object>();
        map.put("uid",6);
        map.put("uname","map");
        map.put("upwd","999");
        mapper.addUser(map);
        sqlSession.commit();
        sqlSession.close();
    }
