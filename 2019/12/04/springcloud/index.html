SpringCloud

发表于 2019-12-04 更新于 2020-02-17 分类于 笔记

Spring Cloud为开发人员提供了快速构建分布式系统中一些常见模式的工具（例如配置管理，服务发现，断路器，智能路由，微代理，控制总线）。分布式系统的协调导致了样板模式, 使用Spring Cloud开发人员可以快速地支持实现这些模式的服务和应用程序。他们将在任何分布式环境中运行良好，包括开发人员自己的笔记本电脑，裸机数据中心，以及Cloud Foundry等托管平台

服务注册与发现Eureka

为了方便管理各个module，可以先建立父maven工程

服务注册中心

  1. 直接Spring Initiallizr创建

选择Cloud Discovery->Eureka Server

  1. 在启动类里添加注解
@SpringBootApplication
@EnableEurekaServer
public class EurekaServerApplication {
    public static void main(String[] args) {
        SpringApplication.run(EurekaServerApplication.class, args);
    }
}
  1. 编写配置文件
spring:
  application:
    name: eureka-server

server:
  port: 7000

eureka:
  instance:
    hostname: localhost
  client:
    register-with-eureka: false
    fetch-registry: false
    service-url:
      defaultZone: http://${eureka.instance.hostname}:${server.port}/eureka/

服务提供与调用Eureka

服务提供者

  1. 新建moudle

选择Eureka Discover Client以及web

  1. 配置文件
spring:
  application:
    name: eureka-producer

server:
  port: 8000

eureka:
  client:
    service-url:
      defaultZone: http://localhost:7000/eureka/

Finchley版本可以不用再注解启动类

  1. 编写controller

提供hello服务

@RestController
@RequestMapping("/hello")
public class HelloController {

    @RequestMapping("/")
    public String hello(@RequestParam String name){
        System.out.println("produce-ok");
        return "Hello, " + name + " " + new Date();

    }
}

服务消费者（Feign）

  1. 新建module

选择web、eureka-client·、OpenFeign

  1. 配置文件
spring:
  application:
    name: eureka-consumer-feign-hystrix

server:
  port: 9003

eureka:
  client:
    service-url:
      defaultZone: http://localhost:7000/eureka/ # 指定 Eureka 注册中心的地址

feign:
  hystrix:
    enabled: true

注意，后面是熔断机制

  1. 在启动类添加注解
@SpringBootApplication
@EnableFeignClients
public class EurekaProducerFeignApplication {

    public static void main(String[] args) {
        SpringApplication.run(EurekaProducerFeignApplication.class, args);
    }

}
  1. 定义一个接口

绑定服务提供者的接口

@FeignClient(name = "eureka-producer",fallback = HelloRemoteHystrix.class)
public interface HelloRemote {

    @GetMapping("/hello/")
    String hello(@RequestParam(value = "name") String name);
}

注意：

  • fallback是熔断机制的内容
  • 必须和服务提供者中的controller里的方法名和参数一致
  1. 编写controller

将刚定义的那个接口注入到controller里，然后像普通方法一样去调用即可。

@RequestMapping("/hello")
@RestController
public class HelloController {

    @Autowired
    HelloRemote helloRemote;

    @GetMapping("/{name}")
    public String index(@PathVariable("name") String name){
        return helloRemote.hello(name+"!");
    }
}

熔断机制 Feign Hystrix

熔断只作用在服务的调用这一端，所以只需要改动eureka-consumer-feign moudle的代码即可。

实际上，上边的笔记里，已经改了。这里再细说一下。

  1. pom依赖

不用改动

  1. 配置文件
spring:
  application:
    name: eureka-consumer-feign-hystrix

server:
  port: 9003

eureka:
  client:
    service-url:
      defaultZone: http://localhost:7000/eureka/ # 指定 Eureka 注册中心的地址

feign:
  hystrix:
    enabled: true
  1. 创建回调类
@Component
public class HelloRemoteHystrix implements HelloRemote {

    @Override
    public String hello(@RequestParam(value = "name") String name) {
        return "Hello World!";
    }
}
  1. 修改服务调用类的接口

添加fallback属性

@FeignClient(name = "eureka-producer",fallback = HelloRemoteHystrix.class)
public interface HelloRemote {

    @GetMapping("/hello/")
    String hello(@RequestParam(value = "name") String name);
}
  1. 测试

启动：

  • 服务注册中心
  • 服务提供
  • 服务调用（已经添加了熔断机制）

正常访问->正常显示（加入Hystrix后，不影响正常访问）

手动停止服务提供

再访问->Hello World

再重启服务提供

再访问->正常显示

总结：

使用Hystrix，能防止雪崩，使系统能够自动降级、自动恢复。
